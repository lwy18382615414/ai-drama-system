# Handoff: 小说 → 短剧 工作台（AI 漫剧前端）

## Overview
「小说 → 短剧」制作工作台的完整高保真设计：从项目列表、小说导入，到章节事件抽取、分批规划、集级管线看板、集工作区、资产库与资产详情。设计依据 `frontend-design.md`（随包附上），该文档同时是**功能与交互的权威规格**（状态派生规则、SSE 同步、stale 治理、Job 聚合、错误分类映射），实现时必须一并遵守。

## About the Design Files
本包中的 `.dc.html` 文件是**用 HTML 制作的设计参考原型**（自定义模板运行时，可直接在浏览器打开查看，但不是生产代码）。任务是：**在目标代码库中重新实现这些设计**。目标技术栈已由设计文档指定：**Vue 3 + Vite + Naive UI + TanStack Query（@tanstack/vue-query）+ Pinia**。若从零新建项目，按此栈初始化。

阅读原型源码时：`<x-dc>` 内是模板（`{{ }}` 为数据洞、`sc-for`/`sc-if` 为循环/条件、`style-hover` 为悬浮态），`<script data-dc-script>` 内的 `renderVals()` 返回模板数据与事件处理器；所有演示数据（集列表、章节、资产、Job）都在该函数内，可直接作为 mock 数据使用。`image-slot.js` 是图片占位组件，实现时替换为真实图片/上传组件。

## Fidelity
**High-fidelity**。颜色、字号、间距、圆角、状态色均为最终值，应按像素还原（用 Naive UI 组件承载时以本文 Design Tokens 覆盖其主题变量）。

## Interactions & Behavior
- **依赖门控**：未就绪阶段按钮置灰 + tooltip 显示后端原因；不存在可点但必然失败的入口
- **stale 是状态不是错误**：照常浏览、只阻断下游生成；黄角标必带原因；主按钮「生成」→「重新生成」
- **批量操作两段式**：多选/一键入口 → 确认模态（展示数量，**不展示金额——本设计已明确移除全部计费/费用 UI，勿实现**）→ 创建 Job → 任务抽屉可跟踪
- **乐观更新仅限纯编辑**（剧本文本、image_prompt）；生成类一律等真实状态，禁止假进度
- **SSE**：每项目一条 EventSource，事件只触发 query invalidate（映射表见设计文档 §3.2）；断线降级 10s 轮询 + 顶部横幅
- **错误分类 → 文案/动作映射**：按设计文档 §7 的表实现（content_rejected 不提供自动重试等）
- spinner：`border 2px + border-top-color 主色 + rotate 1s/0.9s linear infinite`；骨架 shimmer：`linear-gradient(90deg,#eef1f3 25%,#f7f9fa 50%,#eef1f3 75%)` 1.4s
- 模态遮罩 rgba(15,18,20,.32)，抽屉遮罩 .28；卡片阴影 0 16px 48px rgba(0,0,0,.2)

## State Management（对应设计文档 §3、§9）
- TanStack Query 管全部服务端状态，query key 见文档 §3.1；Pinia 只存 UI 状态（多选集合、抽屉开合、向导步骤）
- Composables：`useProjectTaskStream` / `usePipelineStatus` / `useJob` / `useCostConfirm`（本设计中确认框已无金额，可简化为 `useBatchConfirm`）
- 组件清单：PipelineBoard、PipelineStageCell、TaskDrawer、StaleBanner、ReplanWizard、BatchConfirmModal、AppearanceTimeline、ErrorHint
